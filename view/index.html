Route description

Method Path Description
get / To render the home page(index.html)
get /users To get the users
get /user/:id To get the user by its id
get /products To retrieve all the products
get /product/:id To retrieve a certain product by specifying its id
post /users(registration) Route encrypts password,saves successfully in DB, and gives back a JWT to the front end.
put /users/:id Route updates successfully (including encrypting password if one is given)
post /users/(login) Route successfully checks for user in DB by email, and compares passwords correctly.JWT is sent back.
delete /users/:id Route updates successfully removes user from DB.
middleware /products All '/product' routes use middleware to make sure that only registered users with a token can see a product's details, or "view detail" on a product.
post /products Successfully saves product in DB.
put /products/:id Route updates successfully (can only update product of logged in user)
put /users/:id Route updates successfully (including encrypting password if one is given)
put /users/:id Route updates successfully (including encrypting password if one is given)
delete /products/:id Route updates successfully (can only delete product of logged in user)
